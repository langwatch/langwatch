Each row is one unimplemented-tagged occurrence. Class is one of:

- `KEEP` — scenario describes intended behavior that still exists; needs a test binding (Phase 3).

- `DUPLICATE` — already covered elsewhere; reference the duplicate in the rationale.

| Class | Count |

| **Total** | **98** |

The model-providers feature set is mature and fully implemented in the source tree
(registry, drawer form, custom models dialog, default-provider section, deletion guards,
onboarding flow, API key validation). The dominant gap is **test coverage**, not
behavioral drift — hence almost everything classifies KEEP. The two DUPLICATE rows are
the "Default Model badge" scenarios duplicated verbatim between `default-provider.feature`
and `provider-list.feature`; the provider-list copies are the canonical home (badges live
on the list page) so the default-provider copies are flagged for removal in Phase 1.
