The `langwatch` CLI is the dev-side surface of LangWatch Governance.
It does three things:

1. Authenticates you against your organization via SSO
   (`langwatch login`).
2. Wraps your existing AI tools so every request flows through your
   organization's gateway with your identity attached
   (`langwatch claude`, `langwatch codex`, etc.).
3. Provides utility commands for inspection and shell integration
   (`langwatch whoami`, `langwatch dashboard`, `langwatch init`).

## Install

<CodeGroup>

```bash macOS / Linux
curl -sSL https://get.langwatch.com | sh
```

```bash Homebrew
brew install langwatch/tap/langwatch
```

```powershell Windows
iwr -useb https://get.langwatch.com/install.ps1 | iex
```

```bash From source
go install github.com/langwatch/langwatch/services/cli@latest
# binary lands at $GOPATH/bin/cli — rename to langwatch
```

</CodeGroup>

## Global behavior

- All commands respect `LANGWATCH_URL` (control plane base URL,
  default `https://app.langwatch.ai`) and `LANGWATCH_GATEWAY_URL`
  (gateway base URL, default `https://gateway.langwatch.ai`).
  Self-hosted users override these at install time.
- Config lives at `~/.langwatch/config.json` (mode 0600).
  Override with `LANGWATCH_CLI_CONFIG=/path/to/config.json`.

## Subcommands

### `langwatch login`

Initiates the [RFC 8628 device-code](https://www.rfc-editor.org/rfc/rfc8628)
OAuth flow. Opens your default browser to a verification URL,
delegates to your organization's IdP for SSO, then returns control
to the CLI.

On success, persists access token, refresh token, user identity,
organization metadata, and an auto-issued personal virtual key.

```bash
$ langwatch login
→ Opening browser to authenticate...
→ Verification URL: https://app.langwatch.ai/cli/auth?code=ABCD-EFGH

⠋ Waiting for you to log in
```

### `langwatch logout`

POSTs to `/api/auth/cli/logout` to server-revoke the refresh token,
then clears `~/.langwatch/config.json` locally. Best-effort: if the
server call fails (network down, server returns an error), the
local clear still happens — otherwise "logout" would leave a usable
token on disk. Idempotent — safe to run when not logged in.

```bash
$ langwatch logout
Logged out — credentials cleared.
```

### `langwatch whoami`

Prints the current identity, organization, gateway URL, and personal
VK prefix. Exits non-zero with a clear message if not logged in.

```bash
$ langwatch whoami
User:         jane@miro.com
Name:         Jane Doe
Organization: Miro
Gateway:      https://gateway.langwatch.ai
Dashboard:    https://app.langwatch.ai
Personal VK:  lw_vk_live_01HZ…
```

### Tool wrappers — `claude`, `codex`, `cursor`, `gemini`

Each wrapper:

1. Loads the local config.
2. Computes the env vars the underlying tool needs to talk to the
   gateway:
   - `claude` → `ANTHROPIC_BASE_URL`, `ANTHROPIC_AUTH_TOKEN`
   - `codex` → `OPENAI_BASE_URL`, `OPENAI_API_KEY`
   - `cursor` → both pairs above (Cursor uses both APIs)
   - `gemini` → `GOOGLE_GENAI_API_BASE`, `GEMINI_API_KEY`
3. `exec`s the underlying tool with those env vars merged into the
   inherited environment, replacing the running process on Unix so
   terminal handling, signals, and exit codes match the underlying
   tool exactly.

```bash
# All forwarded args are passed through to the underlying tool:
$ langwatch claude --model claude-3-5-sonnet
$ langwatch codex   "fix the failing tests"
$ langwatch cursor                            # opens Cursor app
$ langwatch gemini  "explain this stack"
```

If the underlying tool isn't on `$PATH`, the wrapper exits with a
clear error pointing at install instructions. The wrapper does
**not** install the underlying tool for you.

### `langwatch shell`

Spawns a subshell with all gateway env vars pre-set, so any tool
you invoke directly inherits them — useful for running scripts that
themselves call AI tooling.

```bash
$ langwatch shell
» Entering langwatch shell (gateway: https://gateway.langwatch.ai). Type `exit` to leave.
$ claude            # already routed through gateway
$ ./my-script.sh    # any AI calls in here go through gateway
$ exit
```

### `langwatch dashboard`

Opens `<control-plane>/me` in your default browser — your personal
usage dashboard.

```bash
$ langwatch dashboard
Opening https://app.langwatch.ai/me
```

### `langwatch init <shell>`

Prints an `eval`-able snippet so your shell auto-exports the gateway
env vars. Use this when you'd rather run plain `claude` / `codex`
than the wrappers.

Supported shells: `zsh`, `bash`, `fish`, `cmd`, `powershell` (alias
`pwsh`). Defaults to `zsh`.

<CodeGroup>

```bash zsh / bash
# In ~/.zshrc or ~/.bashrc
eval "$(langwatch init zsh)"
```

```fish fish
# In ~/.config/fish/config.fish
langwatch init fish | source
```

```powershell PowerShell
# In $PROFILE
langwatch init powershell | Invoke-Expression
```

```cmd Windows cmd
:: Run on shell start
langwatch init cmd > %TEMP%\lw.cmd && %TEMP%\lw.cmd
```

</CodeGroup>

After re-sourcing, plain `claude` / `codex` / `cursor` / `gemini`
talk to the gateway. The wrappers continue to work too — they're
just shorthand for "set env vars + exec".

### `langwatch version`

Prints the binary version (set at build time via `-ldflags`).

## Token lifecycle

Access tokens are short-lived (~1h). Before any wrapped command
(`langwatch claude`, `codex`, `cursor`, `gemini`, `shell`) runs,
the CLI checks the access token's remaining lifetime; if it's
under 5 minutes, a transparent refresh happens via
`POST /api/auth/cli/refresh`, persisting both rotated tokens. The
underlying tool starts with a fresh token in env every time.

If the server returns 401 to a refresh (the refresh token has been
server-revoked — typically because an admin clicked **Revoke** on
the user, or the user was deactivated in the IdP), the CLI:

1. Deletes `~/.langwatch/config.json`
2. Prints `Session revoked — run `langwatch login` to sign in again`
3. Exits non-zero

Re-running `langwatch login` re-establishes credentials. There is no
silent reuse of revoked state.

## Config file

`~/.langwatch/config.json`:

```json
{
  "gateway_url": "https://gateway.langwatch.ai",
  "control_plane_url": "https://app.langwatch.ai",
  "access_token": "lw_at_…",
  "refresh_token": "lw_rt_…",
  "expires_at": 1714248000,
  "user": {"id": "u_…", "email": "jane@miro.com", "name": "Jane Doe"},
  "organization": {"id": "o_…", "slug": "miro", "name": "Miro"},
  "default_personal_vk": {
    "id": "vk_…",
    "secret": "lw_vk_live_…",
    "prefix": "lw_vk_live_01HZ"
  }
}
```

The file is created with mode `0600`. Treat it like a private SSH
key — anyone with read access can act as you against the gateway.

## REST API surface

For self-hosting, custom CLI clients, or anyone integrating against
the device-flow directly, the wire surface is:

| Endpoint | Purpose | Success | Notable error states |
|---|---|---|---|
| `POST /api/auth/cli/device-code` | Mint a device-code + user-code pair | `200` JSON `{ device_code, user_code, verification_uri, verification_uri_complete, expires_in, interval }` | `400 invalid_request` |
| `POST /api/auth/cli/exchange` | Long-poll for approval | `200` JSON `{ access_token, refresh_token, expires_in, refresh_expires_in, user, organization, default_personal_vk }` | `428 authorization_pending` (keep polling), `408 expired_token`, `410 access_denied`, `429 slow_down` (re-poll too fast — back off) |
| `POST /api/auth/cli/refresh` | Rotate access token | `200` JSON `{ access_token, refresh_token, expires_in, refresh_expires_in }` (refresh-token rotation — old token is invalidated) | `401 invalid_grant` (CLI must wipe local state) |
| `POST /api/auth/cli/logout` | Revoke a refresh-token server-side | `200 { ok: true }` (idempotent) | — |
| `GET /api/auth/cli/lookup?user_code=...` | Browser-side: verify a pasted user-code | `200 { user_code, status, created_at, expires_at }` | `401 unauthorized` (user must sign in first), `404 not_found`, `410 expired` |
| `POST /api/auth/cli/approve` | Browser-side: user clicks "Approve" | `200 { ok, personal_vk_label, organization_id }` | `401 unauthorized`, `403 forbidden` (caller not in target org), `409 already_resolved` |
| `POST /api/auth/cli/deny` | Browser-side: user clicks "Deny" | `200 { ok }` (idempotent) | `401 unauthorized` |

Wire format is **snake_case JSON** (matches RFC 8628 + every other
OAuth library). Origin enforcement applies to all endpoints — pass
`Origin: <your-base-url>` from any client outside a browser.

The browser approval surface is `https://<your-base-url>/cli/auth?user_code=XXXX-YYYY`
(the `verification_uri_complete` field). Unauthenticated visitors
are bounced through SSO and return to the page automatically.

### Verified behavior

The reference server passes all of:

- Each `device-code` mint returns a unique `user_code` and `device_code`
- `user_code` is 8 chars, base32 alphabet (no I/O/0/1/L/U), dashed `XXXX-YYYY`
- `verification_uri` ends with `/cli/auth`
- `expires_in` defaults to 600s (configurable), `interval` defaults to 5s
- Polling `/exchange` faster than the per-device rate-limit (4s) returns `429 slow_down`
- Polling with an unknown / TTL-expired `device_code` returns `408 expired_token`
- `/refresh` with an unknown / expired token returns `401 invalid_grant`
- All browser-side endpoints (`/lookup`, `/approve`, `/deny`) require an
  active session cookie; unauthenticated requests get `401 unauthorized`
  with a body the CLI surfaces verbatim
- `/logout` is idempotent — passing an unknown / already-revoked refresh
  token still returns `200 { ok: true }`

## Troubleshooting

<AccordionGroup>
  <Accordion title="Browser didn't open at login">
    The CLI prints the verification URL — copy it into your browser
    manually. The fallback message also includes the user code so you
    can paste it on the verification page.
  </Accordion>
  <Accordion title="`Tool not found in PATH`">
    The wrapper expects the underlying tool (Claude Code, Codex,
    etc.) to be installed already. Install it with the official
    installer, then retry the wrapper.
  </Accordion>
  <Accordion title="`not logged in — run langwatch login`">
    Either you've never run login on this machine, or your refresh
    token has been revoked. Re-run `langwatch login`.
  </Accordion>
  <Accordion title="`Budget limit reached`">
    The gateway is enforcing your monthly cap. Ask your admin to
    raise it or wait for the period reset. The error message
    includes your admin's email.
  </Accordion>
  <Accordion title="Self-hosted setup — overriding URLs">
    Set `LANGWATCH_URL` (control plane) and `LANGWATCH_GATEWAY_URL`
    (gateway) before running `langwatch login`. The values are
    persisted in the config file on first save.
  </Accordion>
</AccordionGroup>
