The `langwatch` CLI lets you provision and manage AI Gateway resources without the UI. It hits the same public REST API (`/api/gateway/v1/*`) that powers the dashboard — the CLI and UI share a service layer server-side, so behaviour is identical.

Ships in the LangWatch TypeScript SDK npm package: `npm i -g langwatch`.

## Authentication

```bash
langwatch login
```

Paste a personal-access token or a service-account token when prompted. The token is stored under `~/.langwatch/config.json` and used for every subsequent CLI call. To use a different token for a one-off call, set `LANGWATCH_API_KEY=...` in the environment.

Permissions required on the token mirror the RBAC grants documented in [RBAC](/ai-gateway/rbac):

| Command group | Permission |
|---|---|
| `virtual-keys list|get` | `virtualKeys:view` |
| `virtual-keys create` | `virtualKeys:create` |
| `virtual-keys update` | `virtualKeys:update` |
| `virtual-keys rotate` | `virtualKeys:rotate` |
| `virtual-keys revoke` | `virtualKeys:delete` |
| `gateway-budgets *` | `gatewayBudgets:*` |
| `gateway-providers *` | `gatewayProviders:*` |

## Virtual keys

### List

```bash
langwatch virtual-keys list
langwatch virtual-keys list --format json
```

Output (table):

```
Name          Prefix            Env   Status   Providers           Last Used           Created
prod-key      lw_vk_live_01HZX… live  active   openai,anthropic    2026-04-18 21:03    2026-04-10
dev-key       lw_vk_test_01HZW… test  active   openai              2026-04-18 18:21    2026-04-12
stale-key     lw_vk_live_01HZV… live  revoked  openai              2026-03-29 10:15    2026-03-12
```

### Create

```bash
langwatch virtual-keys create \
  --name prod-key \
  --env live \
  --provider gpc_01HZX... \
  --provider gpc_01HZY...
```

`--provider` takes a **provider-credential id** (from `langwatch gateway-providers list`), not a plain provider name. Pass the flag multiple times to bind more than one provider.

Returns the full secret **once**:

```
✓ Created virtual key "prod-key"

⚠  Save the secret below NOW. It will not be shown again.

  lw_vk_live_01HZX9K3MABCDEFGH...JIKLMN0Z

Use it as the API key in OpenAI-compatible clients:
  export OPENAI_API_KEY="lw_vk_live_01HZX..."
  export OPENAI_BASE_URL="https://gateway.langwatch.ai/v1"

Virtual key id: vk_01HZX9K3M...
Prefix:         lw_vk_live_01H...NZ0Z
```

Use `--format json` for scripting — the JSON branch prints `{virtual_key, secret}`:

```bash
VK=$(langwatch virtual-keys create --name ci --provider gpc_01HZX... --format json | jq -r .secret)
```

### Get details

```bash
langwatch virtual-keys get vk_01HZX9K3MN...
```

Prints the VK's ID, name, description, environment, status, prefix, owning principal, bound provider-credential ids, timestamps, and the full `config` JSON (model_aliases, models_allowed, fallback, cache mode, guardrails, blocked_patterns).

### Rotate

```bash
langwatch virtual-keys rotate vk_01HZX9K3MN...
```

Displays the new secret exactly once. The old secret stops working immediately — there is no grace window on the control plane, though gateway nodes may honour the old secret for up to ~60 seconds until their `/changes` feed catches up.

### Revoke

```bash
langwatch virtual-keys revoke vk_01HZX9K3MN...
```

Hard revocation, no grace. Gateway caches invalidate within 60 seconds via `/changes`.

## Gateway budgets

```bash
# List
langwatch gateway-budgets list

# Create (scope + window + limit are all required)
langwatch gateway-budgets create \
  --name eng-team-monthly \
  --scope team --team team_01HZ... \
  --window month --limit 5000 --on-breach warn

# Other scope flags map by kind:
langwatch gateway-budgets create --scope organization --organization org_... --window day --limit 1000 --name "org daily"
langwatch gateway-budgets create --scope project      --project      proj_... --window day --limit 100  --name "project daily"
langwatch gateway-budgets create --scope virtual-key  --virtual-key  vk_...   --window hour --limit 5   --name "vk hourly"
langwatch gateway-budgets create --scope principal    --principal    user_... --window week --limit 50  --name "weekly personal"

# Archive (stops enforcement; history preserved)
langwatch gateway-budgets archive bgt_01HZ...
```

`--window` accepts `minute|hour|day|week|month|total`. `--on-breach` is `block` (default) or `warn`. `--limit` is USD (integer or decimal). `--timezone` (optional IANA string, e.g. `Europe/Amsterdam`) sets when daily/weekly/monthly windows reset.

List output shows spent vs limit as a colourised percentage — red at ≥100%, yellow at ≥80%, green otherwise.

## Provider credentials (gateway bindings)

```bash
# List bindings for the current project
langwatch gateway-providers list

# Bind an existing model-provider to the gateway
langwatch gateway-providers create \
  --model-provider mp_01HZ... \
  --slot primary \
  --rate-limit-rpm 10000 \
  --rate-limit-tpm 1000000 \
  --rotation-policy manual

# Disable a binding (stops routing; preserves audit trail)
langwatch gateway-providers disable gpc_01HZ...
```

Returns the `gpc_*` id for the binding, which you then pass to `langwatch virtual-keys create --provider <gpc_id>`.

`--slot` is free-text (e.g. `primary`, `eu-region`, `canary`). `--rotation-policy` is one of `auto`, `manual`, `external_secret_store`. `--fallback-priority <n>` sets the global fallback ordering (lower = tried first).

List output colourises health: green `healthy`, yellow `degraded`, red `unhealthy`.

## Dogfooding loop

After you've bound a provider and minted a VK, call the gateway end-to-end:

```bash
GPC=$(langwatch gateway-providers create --model-provider mp_01HZ... --slot primary --format json | jq -r .id)
VK=$(langwatch virtual-keys create --name dogfood --provider "$GPC" --format json | jq -r .secret)

curl -s -X POST https://gateway.langwatch.ai/v1/chat/completions \
  -H "Authorization: Bearer $VK" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-5-mini","messages":[{"role":"user","content":"ping"}],"max_tokens":8}' | jq
```

Inspect response headers for correlation:

```bash
curl -sD- -o/dev/null -X POST https://gateway.langwatch.ai/v1/chat/completions \
  -H "Authorization: Bearer $VK" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-5-mini","messages":[{"role":"user","content":"ping"}],"max_tokens":8}' | \
  grep -iE '^(x-langwatch-|traceparent|x-request-id)'
```

You'll see `x-langwatch-trace-id`, `x-langwatch-span-id`, `traceparent`, and `x-langwatch-request-id` — see [Observability](/ai-gateway/observability).

## Scripting with `--format json`

Every command accepts `--format json`. The default is `text` (pretty) for single-object commands and `table` for `list`.

```bash
# Archive every budget that's already hit its cap
langwatch gateway-budgets list --format json | \
  jq -r '.[] | select((.spent_usd | tonumber) >= (.limit_usd | tonumber)) | .id' | \
  xargs -n1 -I{} langwatch gateway-budgets archive {}

# Revoke every active test VK older than 90 days
langwatch virtual-keys list --format json | \
  jq -r '.[] | select(.environment == "test" and .status == "ACTIVE" and .created_at < "2026-01-18") | .id' | \
  xargs -n1 -I{} langwatch virtual-keys revoke {}
```

## Environment variables

| Variable | Meaning |
|---|---|
| `LANGWATCH_API_KEY` | Personal-access / service-account token. Overrides `~/.langwatch/config.json`. |
| `LANGWATCH_PROJECT` | Default project id for commands that require one. |
| `LANGWATCH_ENDPOINT` | LangWatch API base URL (for self-hosted). Default `https://app.langwatch.ai`. |
| `LANGWATCH_NO_COLOR` | Disable ANSI colours in output. |

## Exit codes

- `0` — success.
- `1` — user error (missing token, invalid flags, permission denied, validation errors).
- `2` — upstream error (network, control-plane outage).

Use `--verbose` to see full HTTP request/response details when debugging.
