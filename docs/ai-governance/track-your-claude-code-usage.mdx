This is the version of LangWatch for one person tracking one Claude Code seat. No org rollout, no IT admin. You sign up, you wire Claude Code, you see your own spend on `/me`.

How you pay Anthropic decides which path you walk:

| Your setup | Path | Jump to |
|---|---|---|
| Pay-per-token Anthropic API key | API key path (gateway with spend control) | [Path 1: API key](#path-1-api-key) |
| Claude Max / Pro subscription (5x or 20x) | Subscription path (OTLP ingestion, no key required) | [Path 2: Claude plan](#path-2-claude-plan) |

If you have both, wire the API key path first then add the subscription path on top. Both write to the same `/me` page.

When you finish you will have:

- A LangWatch project bound to your account.
- Claude Code wired to LangWatch through a single command (`langwatch claude`).
- A live `/me` dashboard showing your monthly spend, request count, most-used model, and per-tool cost breakdown.

<Info>**Pairs with:** [Personal AI Tools Portal](/ai-governance/personal-portal/overview) (the `/me` page you will be living on).</Info>

## Step 0: Get LangWatch (both paths)

Pick one and stop on `/me`:

**SaaS (fastest).** Sign up at [app.langwatch.ai](https://app.langwatch.ai). The onboarding flow creates a personal organization, a workspace, and a project. You land on `/me` ready to go.

**Self-hosted (one command).**

```bash
npx @langwatch/server
```

The CLI bootstraps Postgres, ClickHouse, Redis, and the app on `http://localhost:5560`. Open it in your browser, sign up with email + password, and you land on `/me`.

## Path 1: API key

You pay Anthropic per request with an API key. You want full server-side capture of every Claude Code call, exact cost per request, and the ability to block calls when you cross a budget.

This path routes every Claude Code request through the LangWatch AI Gateway. LangWatch holds your Anthropic key, swaps it in at request time, captures the full request and response server-side, and writes a trace.

### 1.1 Add your Anthropic credential

On `/me`, scroll to **Model providers (issue your own virtual key)**. Click the **Anthropic** tile to open the provider drawer, then paste your Anthropic API key.

<Frame caption="The Model providers section of /me. Click the Anthropic tile to open the provider drawer.">
  ![Model providers section](/images/ai-governance/post-bypass/10-me-trace-ingest-scrolled.png)
</Frame>

This credential never leaves LangWatch. The Virtual Key you mint next is what Claude Code will see; LangWatch holds the upstream key.

After saving the credential, the drawer mints a Virtual Key for you (prefix `vk-lw-`) and shows it once.

<Frame caption="The Anthropic provider drawer after a Virtual Key is issued. Copy the vk-lw- value once; the langwatch CLI will reuse it automatically once you log in.">
  ![Virtual Key issued](/images/ai-governance/post-bypass/36-vk-rotated-secret-reveal.png)
</Frame>

<Warning>The secret half of the Virtual Key is shown once. If you lose it, re-mint from the same tile.</Warning>

### 1.2 Install the LangWatch CLI

In your terminal:

```bash
npm install -g langwatch
```

### 1.3 Log in

```bash
langwatch login --device
```

A browser tab opens at LangWatch with a one-time approval code. Click **Approve**; the CLI receives a long-lived refresh token plus your default Virtual Key, saved at `~/.langwatch/config.json`.

<Frame caption="The terminal flow for `langwatch login --device`. Browser approval pairs the CLI with your account and binds your default Virtual Key.">
  ![langwatch login device flow](/images/ai-governance/cli-flow/01-cli-login-device-flow.png)
</Frame>

### 1.4 Run Claude Code through LangWatch

```bash
langwatch claude
```

The wrapper detects your Virtual Key, sets `ANTHROPIC_BASE_URL` + `ANTHROPIC_AUTH_TOKEN` on the spawned `claude` process, and hands off the terminal to Claude Code as normal. Every API call from this session routes through the gateway and lands as a trace on `/me`.

<Frame caption="A `langwatch claude` session in tmux. The wrapper injects the gateway env, then spawns claude with stdio inherited so the experience is identical to vanilla claude code.">
  ![langwatch claude tmux](/images/ai-governance/cli-flow/06-tmux-langwatch-claude-real.png)
</Frame>

### 1.5 See the trace on /me

Refresh `/me`. Within seconds you should see:

- A non-zero number in **Spent this month**.
- A row in **Recent activity** with the model, token counts, and dollar cost.
- A model breakdown under **By tool**.

<Frame caption="My Usage panel populated by a Claude Code session routed through the gateway. Exact cost per call, per-model breakdown, daily spend chart.">
  ![My Usage populated](/images/ai-governance/post-bypass/37-me-usage-scrolled.png)
</Frame>

Open **Traces** in the sidebar for the per-request detail. Every Claude Code turn shows as a `gen_ai.*` span with prompts, responses, latency, and cost.

<Frame caption="Trace detail for a single Claude Code turn captured through the gateway. Prompt, assistant response, model, tokens, cache_creation / cache_read split, and the exact dollar cost.">
  ![Path 1 trace detail](/images/ai-governance/cli-flow/07-trace-detail-pong-via-gateway.png)
</Frame>

Done. Every future `langwatch claude` session writes to the same `/me`. Skip to [What's next](#whats-next).

## Path 2: Claude plan

You pay Anthropic flat per month and have a Claude Max seat (5x or 20x). Anthropic does not give you per-request cost from their billing surface; the only way to see your actual usage is to capture Claude Code's own OpenTelemetry emissions and let LangWatch normalise them.

This path does not require a Virtual Key. The `langwatch claude` wrapper falls back to OTLP ingestion, mints a personal ingestion binding under the hood, and wires the OTLP env block onto the spawned `claude` process.

### 2.1 Install the LangWatch CLI

```bash
npm install -g langwatch
```

### 2.2 Log in

```bash
langwatch login --device
```

Same flow as Path 1: browser approval pairs the CLI with your account.

<Frame caption="The terminal flow for `langwatch login --device`. No API key needed for Path 2; the CLI just needs your LangWatch identity.">
  ![langwatch login device flow](/images/ai-governance/cli-flow/01-cli-login-device-flow.png)
</Frame>

### 2.3 Run Claude Code through LangWatch

```bash
langwatch claude
```

Because no Virtual Key is present in your config, the wrapper auto-installs a personal Claude Code ingestion binding on your behalf, mints a one-time `ik-lw-` token, sets the OTLP env block (`CLAUDE_CODE_ENABLE_TELEMETRY`, `OTEL_LOG_USER_PROMPTS`, `OTEL_EXPORTER_OTLP_*`) on the spawned process, and hands off the terminal.

You keep paying Anthropic flat. Your Claude Code session keeps using your Claude Max plan. LangWatch reads the OTel log records Claude Code emits (model, input_tokens, output_tokens, cache_creation_tokens, cache_read_tokens, cost_usd, prompt body) and renders them on `/me`.

<Frame caption="A `langwatch claude` session running under the Claude plan path. Same wrapper UX as Path 1; the difference is the wrapper switched to OTLP ingestion mode automatically because no Virtual Key was on file.">
  ![langwatch claude tmux path 2](/images/ai-governance/cli-flow/08-tmux-claude-pathb-otel-real.png)
</Frame>

### 2.4 See the trace on /me

Refresh `/me`. Within 30 seconds:

- A non-zero number in **Spent this month** (computed from Claude Code's own `cost_usd` field).
- A row in **Recent activity** per Claude Code turn.
- A model breakdown under **By tool**.

<Frame caption="My Usage panel populated by a Claude plan session. Same surface as Path 1; the cost number comes from Claude Code's own usage event rather than from the gateway.">
  ![My Usage populated](/images/ai-governance/post-bypass/37-me-usage-scrolled.png)
</Frame>

Open **Traces** in the sidebar for the per-request detail.

<Frame caption="Trace detail for a Claude plan session. Model, tokens, cache_creation_tokens, cache_read_tokens, cost_usd, and prompt body all rendered from Claude Code's OTel log records.">
  ![Path 2 trace detail](/images/ai-governance/post-bypass/14-claude-pathb-trace-detail-storegate-fix.png)
</Frame>

<Info>Claude Code 2.1.x does not emit the assistant's response text in OTel, only token counts. The input prompt and full cost / token breakdown render in `/me/traces`; the output text stays empty until Anthropic ships it on their OTel emission.</Info>

## What's next

The two paths land on the same surface. The differences below all live on `/me`.

### Set a personal budget

`/me` → **Settings** → **Personal budget**. Set a monthly dollar cap.

- **Path 1 (API key)**: the gateway enforces the cap. Requests fail with `budget_exceeded` once you cross.
- **Path 2 (Claude plan)**: you get an alert at 80% and 100%. The actual Claude Code calls keep flowing; Anthropic does not give the receiver a way to gate them.

### Add anomaly rules (admin scope)

Anomaly rules live on the org-admin surface, not `/me`, since they evaluate across every member's spend. Solo users on a personal org have admin scope by default, so head to `/governance/anomaly-rules` to pick a metric (cost per day, spike vs baseline, token usage by model) and a threshold. See [Anomaly Rules](/ai-governance/anomaly-rules) for the full schema.

### Add other assistants

`langwatch <tool>` is wired for the other supported assistants on the same two-path logic: Virtual Key on file = gateway, no key = OTLP ingestion.

| Command | Path 1 (Virtual Key on file) | Path 2 (no key) |
|---|---|---|
| `langwatch claude` | Routes via gateway with `ANTHROPIC_BASE_URL` + `ANTHROPIC_AUTH_TOKEN` | OTLP ingestion via `CLAUDE_CODE_ENABLE_TELEMETRY` + `OTEL_LOG_USER_PROMPTS` |
| `langwatch codex` | Routes via gateway with `OPENAI_BASE_URL` + `OPENAI_API_KEY` | OTLP ingestion via `[otel]` block written to `~/.codex/config.toml` |
| `langwatch gemini` | Routes via gateway with `GOOGLE_GEMINI_BASE_URL` + `GEMINI_API_KEY` | OTLP ingestion (gen_ai.* native) |
| `langwatch opencode` | Routes via gateway with both `OPENAI_*` + `ANTHROPIC_*` env pairs | OTLP ingestion; opencode 1.14 emits structural spans only (no `gen_ai.*` attrs yet, vendor upstream gap) |

The two paths are mutually exclusive per session per tool. The wrapper picks one for you based on whether a Virtual Key is present.

## Privacy

Everything you mint here is scoped to your personal project. If you signed in to LangWatch via a shared org workspace, switch to your personal workspace before running `langwatch login` so the wrapper binds to the right account; the workspace picker is at the top of the sidebar.

For self-hosted (`npx @langwatch/server`), data never leaves your machine. The Postgres, ClickHouse, and Redis containers all bind to localhost.

## Troubleshooting

**Nothing in Recent activity after 30 seconds.** Confirm `~/.langwatch/config.json` has a `default_personal_vk` field (Path 1) or that the wrapper printed an `ik-lw-` token mint line on the first run (Path 2). If neither is true, re-run `langwatch login --device` and then `langwatch claude` once before inspecting `/me`. `langwatch init-shell` prints the eval-able shell snippet showing exactly which env vars the wrapper would inject if you want to inspect them directly.

**401 from the gateway (Path 1).** Your Virtual Key on `~/.langwatch/config.json` may have rotated. Run `langwatch login --device` again, or re-mint the Virtual Key from `/me` → Model providers and re-run `langwatch login`.

**Cost shows $0 even though traces land (Path 2).** The model alias in the span does not match a current entry in LangWatch's cost catalog. Check the span's `gen_ai.request.model` attribute and confirm it is a current-gen identifier (`claude-sonnet-4-6`, `claude-opus-4-7`, `claude-haiku-4-5`). Legacy aliases (`claude-3-5-sonnet-20241022`) silently produce NULL cost.

**Self-hosted: app starts but `/me` is empty.** Make sure you signed up with email + password (the SaaS SSO providers are disabled by default in `npx @langwatch/server`). If you signed up but the page is blank, check `docker logs` on the langwatch container for migration errors.

For deeper failure modes see [CLI debug](/ai-governance/cli-debug) and the [Ingestion Templates reference](/ai-governance/ingestion-templates).
